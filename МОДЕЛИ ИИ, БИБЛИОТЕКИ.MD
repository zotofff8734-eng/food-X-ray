# Модели ИИ и Библиотеки: Food X-Ray

Этот документ — наша база знаний по всем технологиям искусственного интеллекта и ключевым библиотекам, которые мы использовали, рассматривали или планируем использовать в проекте.

## 🚀 Предлагаемый конвейер: Трёхступенчатый анализ

Вместо одной модели предлагается построить конвейер из трёх специализированных шагов для максимальной точности.

### Этап 1: Сегментация (Выделение контуров еды)

- **Модели:** **GroundingDINO + SAM (Segment Anything Model)**
    - **Описание:** Связка двух моделей от Meta. GroundingDINO находит объекты на фото по текстовому описанию (например, "еда"), а SAM строит их точные контуры (маски). Это позволяет отделить еду от тарелки, стола и фона.
    - **Цена:** Бесплатно (Open Source).
    - **Доступность:** Модели и код в открытом доступе.
    - **Сложность внедрения:** **Высокая.** Требует значительных вычислительных ресурсов (мощная GPU с большим объемом VRAM, >12 ГБ). Интеграция в Node.js-сервер сложна и потребует создания отдельного Python-микросервиса для обработки изображений.
    - **Рекомендация:** **Идеальный вариант для будущей версии Pro.** На текущем этапе слишком сложно в реализации и дорого в поддержке для простого прототипа.

### Этап 2: Классификация и оценка калорий

На этом этапе мы определяем, что за блюдо, и ищем его пищевую ценность.

- **Модель (Классификация):** **Обучение собственной модели (например, на YOLO)**
    - **Описание:** Использование архитектуры YOLO (You Only Look Once) для обучения собственной модели на датасетах еды. YOLO хорошо справляется с быстрым обнаружением нескольких объектов.
    - **Датасеты для обучения:**
        - `Food-101`: 101 категория, 101 000 изображений. Классический датасет.
        - `Open Images V6-Food`: Огромный набор данных от Google.
    - **Цена:** Бесплатно (Open Source). Затраты на обучение могут быть, если использовать облачные GPU.
    - **Доступность:** YOLO и датасеты в открытом доступе.
    - **Сложность внедрения:** **Средняя.** Требуются знания в области машинного обучения для подготовки данных и обучения модели.
    - **Рекомендация:** Долгосрочная цель для повышения точности. На старте проще использовать готовые классификаторы.

- **База данных (Пищевая ценность):** **USDA FoodData Central API**
    - **Описание:** Официальная, авторитетная база данных от Министерства сельского хозяйства США. Содержит информацию о сотнях тысяч продуктов, включая КБЖУ, витамины и минералы.
    - **Цена:** **Бесплатно.**
    - **Доступность:** API открыто для всех. Существуют готовые библиотеки-обертки для Python и других языков.
    - **Сложность внедрения:** **Низкая.** Простая интеграция через стандартные HTTP-запросы.
    - **Рекомендация:** **Использовать как основной источник данных о КБЖU.** Это самый надежный и полный бесплатный источник.

- **Альтернативные API (Пищевая ценность):** Edamam, Nutritionix
    - **Описание:** Коммерческие API с похожим функционалом.
    - **Цена:** Freemium (есть бесплатный тариф с ограничениями на количество запросов в месяц).
    - **Доступность:** Легко зарегистрироваться и получить ключ.
    - **Сложность внедрения:** **Низкая.**
    - **Рекомендация:** Держать в качестве запасных вариантов, если функционала USDA окажется недостаточно.

### Этап 3: Оценка глубины и веса (Depth & Weight Estimation)

Это самый сложный и наукоёмкий этап. Точная оценка веса из 2D-фотографии требует сначала построить "карту глубины" изображения, чтобы понять его объем.

- **Подход:** Сначала используется модель оценки глубины (Depth Estimation) для создания карты глубины. Затем, на основе этой карты и контура еды (из Этапа 1), вычисляется примерный объем, который, с учетом плотности продукта, преобразуется в вес.
- **Сложность внедрения:** **Очень высокая.** Требует двух последовательных сложных моделей (сегментация + оценка глубины) и отдельного микросервиса на Python.

#### Сравнительная таблица моделей оценки глубины

| Модель/Библиотека | Размер / Параметры | Мин. GPU | Рекомендуемый GPU | Цена | Ключевые нюансы |
|---|---|---|---|---|---|
| **UniDepth** | ~8 ГБ VRAM | NVIDIA 8 ГБ VRAM | NVIDIA RTX 4090 (24 ГБ) | 0 ₽ | Сбалансированный вариант. V2 на 30% быстрее. Поддерживает `fp16`. |
| **Marigold** | 1.4 млрд | 16 ГБ VRAM | 24+ ГБ VRAM | 0 ₽ | Самая "прожорливая", но и самая точная. Настоятельно рекомендуется `fp16`. |
| **Depth-Anything-V2 (Small)** | 24.8 млн | 4-6 ГБ VRAM | Игровая карта (RTX 3060) | 0 ₽ | **Лучший выбор для старта.** Самая легкая и быстрая (13.3 мс на RTX 4080). |
| **Depth-Anything-V2 (Giant)** | 1.3 млрд | 24+ ГБ VRAM | NVIDIA A100 (40 ГБ) | 0 ₽ | Требует серьезного "железа". "Съедает" 13-14 ГБ VRAM на разрешении 1024. |

#### 🏆 Окончательная рекомендация для старта

Однозначно рекомендуется остановиться на **Depth-Anything-V2** в версии **Small** или **Base**.
- **Низкий порог входа:** Потребление 4-6 ГБ VRAM делает модель доступной для запуска даже на игровом ПК.
- **Высокая производительность:** Высокая скорость обработки (13-29 мс) обеспечит почти мгновенный ответ бота.
- **Вывод:** Это идеальный баланс производительности и доступности для прототипа, позволяющий реализовать функцию оценки объема без колоссальных затрат на инфраструктуру. UniDepth — хороший запасной вариант.

## 💡 Гибридные подходы и готовые решения

Помимо построения собственного конвейера с нуля, существуют готовые проекты и облачные API, которые могут ускорить разработку или служить надежным запасным вариантом.

### CalAnalyZ (Готовое решение с референсным объектом)
- **Описание:** Русскоязычный Open Source проект (готовый код + Telegram-бот), который реализует оценку объема еды по фотографии с использованием **референсного объекта** (например, вилки или пластиковой карты для масштаба).
- **Цена:** **Бесплатно (Open Source).**
- **Доступность:** Код доступен на GitHub.
- **Сложность внедрения:** **Низкая (для старта).** Можно запустить готового бота. **Средняя (для интеграции)**, если требуется встроить его логику в наш существующий проект.
- **Требования к железу:** **Минимальные.** Работает на обычном CPU, не требует GPU.
- **Рекомендация:** **Идеальная стартовая точка для реализации оценки веса.** Дает более точный результат, чем модели глубины без референса, и не требует затрат на инфраструктуру. Главный компромисс — требует от пользователя дополнительного действия (положить карту/вилку рядом с едой).

### Облачные API (Платно, но без забот)

Надежный "план Б" или способ быстро добавить премиум-функции.

- **LogMeal:**
    - **Описание:** Коммерческий сервис, предоставляющий мощное API для распознавания еды, включая оценку веса.
    - **Цена:** Платно, но есть **бесплатный тариф на 20 фото в день**.
    - **Рекомендация:** Отличный вариант для бэкапа или для пользователей с платной подпиской в будущем.

- **FatSecret API:**
    - **Описание:** Популярная база данных пищевой ценности с API.
    - **Цена:** **Есть бесплатный тариф для стартапов.**
    - **Доступность:** **Ограничен рынком США.** Для использования в других странах, вероятно, потребуются обходные пути (например, VPN на сервере).
    - **Рекомендация:** Хороший источник данных, но с географическими ограничениями.

## ✅ Текущий стек (MVP)

- **Модель:** **`nateraw/food` (на базе Food-101)**
    - **Описание:** Готовая модель классификации изображений, натренированная на датасете Food-101. Определяет одно из 101 блюд на фото.
    - **Цена:** Бесплатно (через Hugging Face Inference API).
    - **Доступность:** Легко доступна. Однако бесплатный инференс "засыпает" и может долго отвечать на первый запрос ("cold start").
    - **Сложность внедрения:** **Низкая.** Интеграция через библиотеку `@huggingface/inference`.
    - **Рекомендация:** **Идеальный вариант для прототипа.** Позволяет быстро получить работающий продукт с приемлемой точностью на популярных блюдах.

## 🤔 Рассмотренные ранее (Архив)

- **Модель:** **Google Gemini API**
    - **Описание:** Мультимодальная модель от Google, способная анализировать изображения и текст, выдавая структурированный ответ (JSON) с КБЖУ.
    - **Цена:** Есть бесплатный уровень, но он ограничен.
    - **Доступность:** **Проблемная.** API блокируется при запросах с европейских IP-адресов (где находятся хостинги Render/Railway). Также были блокировки на уровне аккаунта из-за региональных санкций.
    - **Сложность внедрения:** **Низкая.** Удобная библиотека `@google/generative-ai`.
    - **Рекомендация:** **Не использовать.** Несмотря на мощность, модель ненадежна для нашего проекта из-за проблем с доступностью.

- **Модель:** **`Salesforce/blip-image-captioning-large`**
    - **Описание:** Модель для генерации текстового описания по изображению (Image-to-Text).
    - **Цена:** Бесплатно.
    - **Доступность:** **Больше не поддерживается** на бесплатном Serverless Inference API от Hugging Face.
    - **Сложность внедрения:** Низкая.
    - **Рекомендация:** **Не использовать.** Модель устарела и недоступна для нашего стека.

## Сравнительная таблица датасетов для Food AI

### Foodvision Bench/mini-215
*   **Объём / Ключевое содержание:** 215 эталонных блюд (калории, макросы, вес), 40 категорий, 5 кухонь.
*   **Цена:** Бесплатно (прямой бенчмаркинг).
*   **Главная фишка:** Для сравнения систем (Zero-shot или ввод через API). Позволяет оценить MAPE (ошибка калорий).
*   **Сложность интеграции:** Низкая (готовый Python-пакет foodvision-bench).

### JFB (January Food Benchmark)
*   **Объём / Ключевое содержание:** 1,000 изображений реальной еды с ручной проверкой (название, ингредиенты, макросы).
*   **Цена:** Бесплатно (лицензия CC-BY-4.0).
*   **Главная фишка:** Оценка VLM: комплексный скор (Overall Score), сравнение узкоспец. моделей vs GPT-4o.
*   **Сложность интеграции:** Средняя (бенчмарк, код доступен на GitHub).

### DiningBench
*   **Объём / Ключевое содержание:** 3,021 блюд, в среднем 5.27 изображений на блюдо. Есть "сложные" негативные примеры и VQA.
*   **Цена:** Бесплатно (исслед. код на GitHub).
*   **Главная фишка:** Многоуровневое тестирование: Классификация -> Оценка питания -> VQA. Выявляет слабости VLM.
*   **Сложность интеграции:** Низкая / Средняя.

### MM-Food-100K
*   **Объём / Ключевое содержание:** 100,000 пар изображение-текст. Поднабор из 1.2 млн образцов.
*   **Цена:** Условно-бесплатно (основная 10% часть открыта, 90% на коммерческой основе).
*   **Главная фишка:** Мультимодальность и мультикультурность: азиатские блюда, решение проблемы ошибки до 76% в оценке калорий.
*   **Сложность интеграции:** Низкая (загрузка с Hugging Face).

### FastFood
*   **Объём / Ключевое содержание:** 84,446 изображений, 908 категорий фастфуда из 10 брендов. Полные аннотации (ингредиенты, КБЖУ).
*   **Цена:** Бесплатно (для исследований).
*   **Главная фишка:** Специализация на фастфуде: анализ стандартизированных блюд с высокой точностью.
*   **Сложность интеграции:** Низкая (структурированный формат).

### NV-Synth (NutritionVerse-Synth)
*   **Объём / Ключевое содержание:** 84,984 фотореалистичных синтетических изображений из 7,082 3D-сцен.
*   **Цена:** Бесплатно (Open Access).
*   **Главная фишка:** Идеальная разметка: автоматические маски, bounding boxes, глубина. Для задач синтеза и сегментации.
*   **Сложность интеграции:** Средняя (особенности работы с синтетическими данными).

### OmniFood8K
*   **Объём / Ключевое содержание:** 8,036 сцен, полный пайплайн: много-вью, ингредиенты, рецепт и видео приготовления.
*   **Цена:** Бесплатно.
*   **Главная фишка:** Целостность: связь визуального вида с процессом приготовления. Моделирование "сырого" и "готового" блюда.
*   **Сложность интеграции:** Средняя / Высокая (мультимодальная).

### FoodBD
*   **Объём / Ключевое содержание:** 3,523 изображений, полигональная разметка 67 категорий. 1,837 снимков с экспертными нутриентами.
*   **Цена:** Платный доступ к части данных (требуется заказ).
*   **Главная фишка:** Тонкая сегментация: полигональная разметка блюд для извлечения формы, объема.
*   **Сложность интеграции:** Высокая (навыки работы с segmentation).

### ITD (Indian Thali)
*   **Объём / Ключевое содержание:** 7,900 изображений, 796 уникальных тарелок, плотные пиксельные маски.
*   **Цена:** Бесплатно (прямая ссылка на скачивание).
*   **Главная фишка:** Плотная сегментация сложных блюд: анализ "тарелок" с 5-10 наложенными друг на друга элементами.
*   **Сложность интеграции:** Высокая (плотная разметка).

### MetaFood3D
*   **Объём / Ключевое содержание:** 637 3D-объектов, 108 категорий, текстуры и RGB-D видео.
*   **Цена:** Бесплатно (форма заявки).
*   **Главная фишка:** Реалистичная 3D-модель: расчет порций и анализ под любым углом с помощью 3D-реконструкции.
*   **Сложность интеграции:** Очень высокая (для 3D/AR/VR).

### FoodSense
*   **Объём / Ключевое содержание:** 66,842 пар человек-изображение (2,987 уникальных фото) с рейтингами и текстовыми дескрипторами.
*   **Цена:** Бесплатно (данные и код).
*   **Главная фишка:** Мультисенсорное восприятие: предсказание вкуса, запаха, текстуры и звука по фото. Связь когнитивистики и ИИ.
*   **Сложность интеграции:** Средняя (особые метрики оценки).

### ReciFine
*   **Объём / Ключевое содержание:** 2.2 млн рецептов с >97 млн аннотированных сущностей (10 типов: инструменты, действия, состояния, время).
*   **Цена:** Бесплатно (бенчмарк и данные).
*   **Главная фишка:** Контролируемая генерация: создание рецептов с заданными параметрами (время, инструменты, статусы).
*   **Сложность интеграции:** Средняя (NLP-компетенции).

### FoodFake-30K
*   **Объём / Ключевое содержание:** 30,000 изображений (15,000 реальных + 15,000 сгенерированных Flux/Z-Turbo). 15 категорий.
*   **Цена:** Бесплатно (CC BY 4.0).
*   **Главная фишка:** Детекция дипфейков еды: выявление ИИ-сгенерированного контента, проверка подлинности.
*   **Сложность интеграции:** Низкая (готов для PyTorch/TensorFlow).

### Nutrition5k (Оригинал)
*   **Объём / Ключевое содержание:** 5,000+ изображений столовой, ~1,000 уникальных, 15-секундное 3D-сканирование с разметкой.
*   **Цена:** Бесплатно (Google Research).
*   **Главная фишка:** Промышленная съемка: сканеры для замера объема, высокоточные данные о порциях (181 GB).
*   **Сложность интеграции:** Средняя / Высокая (для больших данных).

### 💎 Какой датасет выбрать?
Выбор зависит от целей вашего проекта:

*   **Для бенчмаркинга (объективного сравнения):** Foodvision Bench/mini-215 и JFB.
*   **Для сложной сегментации:** FoodBD (полигоны) или Indian Thali Dataset (пиксельные маски).
*   **Для работы с мультикультурной едой:** MM-Food-100K, OmniFood8K и FastFood (узкая специализация).
*   **Для 3D / AR / метавселенных:** NV-Synth (синтетические 2D изображения) или MetaFood3D (реальные 3D-сканы).
*   **Для уникальных задач:** FoodSense (предсказание вкуса), ReciFine (генерация рецептов), FoodFake-30K (детекция дипфейков) и FoodBD с ITD (сегментация).
