# Серверы для разработки и деплоя: Food X-Ray

Этот документ описывает хостинг-платформы, которые мы использовали или рассматривали для развертывания нашего Telegram-бота, а также опции для хостинга исходного кода.

## ✅ Текущий хостинг: Railway.app

**Railway** — это PaaS (Platform as a Service), которая позволяет разворачивать приложения напрямую из GitHub-репозитория. Это наш текущий и рекомендуемый выбор.

- **Плюсы:**
    - **Щедрый бесплатный тариф:** Предоставляет до $5 или 500 часов процессорного времени в месяц, чего более чем достаточно для прототипа.
    - **Деплой из GitHub:** Процесс развертывания полностью автоматизирован. Достаточно сделать `git push`, и Railway сам соберет и запустит новую версию.
    - **Расположение серверов в США:** Критически важный плюс. Это позволяет обходить блокировки Google Gemini API, которые возникают на европейских серверах.
    - **Стабильная работа Polling:** В отличие от Render, Railway не "усыпляет" фоновые процессы так агрессивно, что позволяет боту стабильно работать в режиме опроса (Polling) без необходимости настраивать Webhooks.
    - **Простое управление переменными окружения.**

- **Минусы:**
    - **Возможные "холодные старты":** Как и на любой бесплатной PaaS, приложение может "засыпать" при неактивности, что приводит к задержке при первом запросе.
    - **Доступность из РФ:** Прямой доступ к сайту может быть затруднен, но работа через VPN решает все проблемы. Оплата картами РФ невозможна.

- **Цена:**
    - **Free Tier:** до $5/месяц.
    - **Developer Plan:** от $10/меся- **Итог:** **Лучший выбор для нашего проекта на данном этапе.** Обеспечивает стабильную работу, обходит ключевые блокировки и прост в использовании.

## 🤔 Рассмотренные ранее хостинги (Архив)

### Render.com

Первая платформа, на которой мы пытались развернуть бота.

- **Плюсы:**
    - Очень простой интерфейс и автоматический деплой из GitHub.
- **Минусы:**
    - **Агрессивное усыпление процессов ("sleeping"):** Бесплатный тариф останавливает сервис через 15 минут бездействия, что не подходит для бота в режиме Polling.
    - **Серверы в Европе:** Запросы с европейских IP-адресов блокировались некоторыми AI-сервисами.

### Koyeb

Рассматривался как альтернатива Render для получения американского IP.

- **Минусы:**
    - **Отменили бесплатный тариф** для новых пользователей на момент, когда мы его рассматривали.

## 📦 Хостинг исходного кода

### GitHub
**GitHub** — это не сервер для хостинга приложения, а система контроля версий и основа нашего CI/CD (Continuous Integration/Continuous Deployment) процесса.

- **Роль в проекте:**
    - **Хранилище кода:** Весь исходный код проекта хранится в репозитории.
    - **Источник для деплоя:** Railway подключен к нашему репозиторию и автоматически развертывает новую версию после каждого `git push`.
    - **Ведение документации:** Все наши `.MD` файлы хранятся здесь же.

#### Доступ AI-ассистентов к приватным репозиториям

Приватность репозитория не является препятствием для работы с AI-ассистентами, но требует явного предоставления доступа.

- **Способы доступа:**
    - **Через приложение (OAuth):** Самый безопасный способ. Инструменты вроде **ChatGPT** позволяют подключить GitHub в настройках и выбрать, к каким именно репозиториям (включая приватные) дать доступ.
    - **Через Personal Access Token (PAT):** Создание "ключа" с определенными правами в настройках аккаунта GitHub. Несет риски, если выдать слишком широкие права.
- **Специализированные инструменты:** Существуют сервисы вроде **Repo Cloak**, которые создают анонимизированное "песочное" окружение для работы AI с приватным кодом, не раскрывая его напрямую.

#### Альтернативы GitHub для хостинга кода

- **Публичные хостинги:**
    - **GitLab:** Мощная альтернатива со встроенным CI/CD.
    - **Bitbucket:** Популярен в корпоративной среде.
    - **Codeberg:** Некоммерческая платформа на базе Gitea.
- **Самостоятельный хостинг (Self-hosted):** Для полного контроля над кодом.
    - **Gitea / Forgejo:** Легковесные, бесплатные Git-серверы с открытым исходным кодом, которые можно развернуть на собственном оборудовании.

## 🚀 Альтернативы Railway для хостинга приложения

Выбор зависит от задачи: простота, круглосуточная работа или наличие GPU.

### Лучшие общие альтернативы

- **Render:** Самый простой переход. Главный недостаток — бесплатный веб-сервис "засыпает" после 15 минут бездействия.
- **Fly.io:** Позволяет запускать Docker-контейнеры. Предоставляет до 3 небольших виртуальных машин бесплатно, но требует больше технических знаний. Отлично подходит для проектов, где важна низкая задержка.
- **Koyeb:** Глобальная сеть серверов с поддержкой serverless-функций.

### Лучшие бесплатные варианты для AI-задач (с GPU)

- **Google Cloud Run (Always Free):** Идеально для контейнеров, которые запускаются по запросу. Отлично подходит для моделей на TensorFlow/PyTorch. Бесплатный пакет включает миллионы запросов и значительное количество вычислительного времени.
- **Oracle Cloud (Always Free):** **Самый мощный бесплатный сервис** для круглосуточной работы. Предоставляет 2 ARM-виртуалки (в сумме 4 vCPU и 24 ГБ ОЗУ), чего с запасом хватит для большинства AI-задач, не требующих GPU. Регистрация сложная, но для серьезных проектов это лучший выбор.

### Другие интересные варианты

- **Vercel / Netlify:** Отлично подходят для фронтенда и небольших serverless-функций (API, вебхуки), но не для постоянных бэкенд-процессов.
- **Coolify (Self-Hosted):** Open-source аналог Railway, который можно развернуть на своем сервере для полного контроля.

### Быстрое сравнение

| Сервис | Тип | Лучше всего подходит для |
|---|---|---|
| **Render** | PaaS | Простого и дешевого запуска монолитов. |
| **Fly.io** | PaaS (Docker) | Проектов, где важна низкая задержка. |
| **Google Cloud Run** | Serverless | Сервисов с переменной нагрузкой. |
| **Oracle Cloud (VM)** | IaaS | Тяжелых проектов, требующих 24/7 работы. |
